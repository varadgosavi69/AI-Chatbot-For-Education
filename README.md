# ⚡ QuickDoubt — AI-Powered Study Assistant

QuickDoubt is a web application that helps students learn faster by combining an AI chat tutor with an intelligent PDF notes explainer. Students often struggle to get clear, immediate answers to subject-specific questions or to make sense of dense lecture notes — QuickDoubt solves both problems: it answers questions step-by-step like a patient tutor, and it transforms uploaded PDF notes into simplified explanations plus four types of visual aids (mind map, flowchart, summary table, pie chart) in seconds.

---

## ✨ Features

- **Subject-based AI chat tutor** — Ask questions across Math, Physics, Chemistry, Computer Science, Biology, or General topics. Claude explains concepts step-by-step at your level.
- **Per-subject chat history** — Conversations are saved per-subject to `localStorage` and persist across sessions.
- **PDF notes explainer** — Upload any text-based PDF and get a structured, student-friendly markdown explanation generated by Claude.
- **Visual aids from notes**
  - 🧠 **Mind map** — hierarchical concept tree
  - 🔄 **Flowchart** — process steps with arrows
  - 📋 **Summary table** — key data in rows/columns
  - 🥧 **Pie chart** — proportional breakdown (powered by Recharts)
- **Error handling** — Non-PDF uploads rejected client-side, scanned/image PDFs detected and explained, Claude parse failures show a friendly fallback.
- **Retry on failure** — Chat errors include a Retry button to resend the last question.

## 🚀 Setup & Running Locally

### Prerequisites
- Node.js 18+
- An [Anthropic API key](https://console.anthropic.com)

### 1. Clone the repository

```bash
git clone https://github.com/varadgosavi69/AI-Chatbot-For-Education.git
cd AI-Chatbot-For-Education
```

### 2. Install dependencies

```bash
# Backend
cd server
npm install

# Frontend
cd ../client
npm install
```

### 3. Configure environment variables

```bash
cd server
cp .env.example .env
```

Open `server/.env` and replace the placeholder with your real Anthropic API key:

```
ANTHROPIC_API_KEY=sk-ant-api03-your-key-here
PORT=3001
```

> **Note:** The key must start with `sk-`. The `.env` file is git-ignored and should never be committed.

### 4. Start the backend

```bash
cd server
npm run dev
```

The server starts at `http://localhost:3001`.

### 5. Start the frontend (new terminal)

```bash
cd client
npm run dev
```

The app opens at `http://localhost:5173`. Vite proxies all `/api` requests to the backend automatically — no manual CORS configuration needed in dev.

## 📡 API Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| `POST` | `/api/ask` | Chat with Claude (subject + question + history) |
| `POST` | `/api/notes/upload` | Upload PDF, returns extracted text |
| `POST` | `/api/notes/explain` | Send text to Claude, returns markdown explanation |
| `POST` | `/api/notes/visualize` | Send text to Claude, returns JSON for 4 visual types |
| `GET` | `/api/health` | Server health check |
